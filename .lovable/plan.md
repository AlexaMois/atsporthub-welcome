

## Генерация технической документации проекта

Создам подробный Markdown-документ с полной технической документацией проекта и сохраню его в `/mnt/documents/`.

### Содержание документации

1. **Обзор проекта** — назначение, стек технологий, ссылки
2. **Архитектура** — структура файлов, диаграмма потоков данных
3. **Маршрутизация** — все маршруты, защита, редиректы
4. **Аутентификация** — полный flow входа по телефону, JWT, персистентность сессии
5. **Edge Function `bpium-api`** — все actions, параметры, ответы, rate limiting
6. **Bpium API** — каталоги, маппинг полей, статусы, связанные поля
7. **Компоненты** — описание каждого компонента с пропсами
8. **Состояние (PortalProvider)** — фильтрация, поиск, статистика, привилегированные роли
9. **Файловые утилиты** — скачивание, просмотр, форматы
10. **ИИ-саммари** — flow генерации, кеширование, поддерживаемые форматы
11. **Безопасность** — CORS, rate limiting, JWT, timing-safe comparison
12. **Секреты и конфигурация** — все переменные окружения
13. **Ограничения и известные особенности**

### Реализация

Один файл: `docs/TECHNICAL_DOCUMENTATION.md` (~150-200 строк). Дополнительно копия в `/mnt/documents/` для скачивания.
